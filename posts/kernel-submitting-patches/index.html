    向kernel提交patch

    五月 26, 2024
    技术博客
    Linux

    过了这么长时间一直没有发布任何文章，也不知道写些什么好，其实博客和小说写了很多内容，但是自己总觉得还没写好，唉！但凡能崩出个屁来，也不至于这样啊！！！

    今天就简单介绍一下如何向Linux内核提交patch

    拉取代码 #

    安装git # 

    apt install git git-email
git config --global user.name "John Doe"
git config --global user.email johndoe@example.com

    配置邮箱 #

    vim ~/.gitconfig，去对应邮箱拿smtp授权

    [sendemail]
smtpEncryption = tls
smtpServer = smtp.gmail.com
smtpUser = yourname@gmail.com
smtpServerPort = 587

    拉主支代码 # 

    git clone https://git.kernel.org/pub/scm/linux/kernel/git/torvalds/linux.git

    拉取分支 #

    大多数子系统Maintainer都有自己的树，可以在MAINTAINERS文件中，找到相应的Maintainer及其git地址

    cd linux
git remote add linux-next https://git.kernel.org/pub/scm/linux/kernel/git/next/linux-next.git
git fetch --tags linux-next
git checkout -b mydev next-****

    任何要合并到linux主线的代码都会首先合并到next中，所以咱们先基于next分支修改

    查看tags # 

    git tag -l "next-*" | tail

    定期更新代码 # 

    git checkout master 
git remote update

    订阅邮件列表 #

    如果对某个子系统感兴趣，或想参与讨论，可以去订阅邮件列表，但一定注意网络礼仪和邮件格式

    在发信前，可以先去搜索有关主题的存档，很多已经详细讨论过的事情只记录在邮件列表档案中

    编译 #

    生成文档 # 

    make htmldocs

    也可以使用在线文档，入门先把Development process里面的内容大致看一遍

    生成配置 # 

    make ARCH=x86_64 allyesconfig

    编译 # 

    make -j4 ARCH=x86_64 EXTRA_CFLAGS="-Wmissing-declarations -Wmissing-prototypes"

    生成其他平台，在后面加上对应的交叉编译工具链

    CROSS_COMPILE=toolchain_path

    提交补丁 #

    补丁的内容大致分成

    • docs
    • warnning
    • bug-fix
    • feature

    首先要熟悉kernel编程规范

    咱们刚入门就从文档注释，编码格式和消除警告入手，也可以更好的了解整个工作流程

    如果你什么都不懂，就可以先看看Linux内核新手计划

    如果你想加入内核开发社区并协助完成一些任务，可以访问Linux内核房管员计划

    制作补丁 # 

    git format-patch -s -1

    在提交之前，使用checkpatch.pl检查补丁是否违规，注意patch格式规则

    ./scripts/checkpatch.pl 0001-*

    达到 0 errors, 0 warnings， 就可以准备将它发送给Maintainer了

    发送邮件，等待回复 #

    用get_maintainer.pl来查看对应模块的Maintainer

    ./scripts/get_maintainer.pl 0001-*

    发送邮件前可以先用自己邮箱试一试，确认无误后，再向Maintainer邮箱发送即可，

    git send-email 0001-* --to 收件人邮箱 --cc 抄送人邮箱
